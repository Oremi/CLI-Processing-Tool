# Contributing to CLI-Processing-Tool

Thank you for contributing! This project is small, but we welcome improvements, bug fixes, and tests.

## Getting started

1. Fork the repository.
2. Create a branch for your work:

```bash
git checkout -b feature/my-change
```

3. Make your changes and add tests if applicable.
4. Run tests:

```bash
go test ./...
```

5. Commit your changes with a descriptive message.
6. Push and open a pull request.

## Coding style

- Follow Go idioms (use `gofmt` formatting).
- Keep functions concise and testable.
- Add or update tests in `main_test.go` for behavior changes.

## What to include in PRs

- Problem statement
- Approach and reasoning
- Test coverage details
